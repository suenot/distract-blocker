================================================================
  Privacy practices → Single purpose description
================================================================
Block user-defined distracting websites to help the user focus.
The extension manages a blocklist, a master on/off toggle, and a
day-of-week + time-of-day schedule that controls when blocking
applies.

================================================================
  Privacy practices → Permission justifications
  (one textarea per permission, paste the matching block)
================================================================

--- declarativeNetRequest justification ---
This is the core blocking mechanism. Dynamic rules redirect
main_frame navigation to an in-extension blocked.html page for
each domain the user has added to their blocklist, and block
sub_frame requests to the same domains. Rules are created from
the user's list via chrome.declarativeNetRequest
.updateDynamicRules(). No content scripts are injected. No
remote code is fetched. Chrome enforces the rules natively.

--- storage justification ---
chrome.storage.local is used to persist user-managed settings
locally: the blocklist of distracting domains, the master
enabled toggle state, the schedule (mode + ranges), the chosen
UI language, and the list/text edit-mode preference. Nothing
is synced via chrome.storage.sync, and nothing is sent off the
device.

--- sidePanel justification ---
The entire user interface (master toggle, blocklist editor,
schedule editor, language switcher) lives inside the side
panel that opens when the user clicks the action icon
(chrome.sidePanel.setPanelBehavior({openPanelOnActionClick:
true})). The extension does not open the side panel
automatically or for any other purpose.

--- alarms justification ---
Used to wake the service worker once per minute via
chrome.alarms.create({ periodInMinutes: 1 }) so the extension
can re-evaluate the user's blocking schedule. When the current
time crosses a scheduled range boundary, blocking is enabled or
disabled by updating declarativeNetRequest dynamic rules.
Alarms are not used for any other purpose.

--- Host permission justification ---
Required by declarativeNetRequest's "redirect" action so the
extension can redirect any matched navigation to a local
extension page (blocked.html). The extension never reads,
modifies, or transmits page content — the host permission only
authorizes the redirect action that the user explicitly
configured via their blocklist.
